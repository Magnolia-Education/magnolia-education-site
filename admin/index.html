Setup Guide

One-time configuration to connect the admin to your live site.

How saving works: When you click "Save & Deploy", the admin sends your changes to a Netlify serverless function at /.netlify/functions/update-content. That function commits content.js to GitHub using a server-side token, and Netlify auto-deploys the new version (~30 seconds).
Step 1 — Set Netlify Environment Variables

Go to your Netlify site → Site Settings → Environment Variables, and add:

ADMIN_PASSWORD — the password used to log into this admin panel
GH_TOKEN — a classic GitHub Personal Access Token with repo scope, authorized for the Magnolia-Education org (Settings → Developer settings → Personal access tokens → Classic)
Step 2 — Change Admin Password

Open admin/index.html and find the line marked // CHANGE_THIS. Change the password string there, commit the file. The new password must also be set in Netlify's environment variables as ADMIN_PASSWORD.

Step 3 — Test

Make a small edit to any field, click "Save & Deploy", and watch for the green "Saved!" message. Check your Netlify dashboard to confirm a new deploy triggered.
